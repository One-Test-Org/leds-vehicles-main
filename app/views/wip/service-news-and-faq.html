 {# Release 1.7.0 #}

05/10/2023

Link to Drivers

  • For users with the Drivers entitlement, Registered keeper, insurance policy holder and named drivers appear as links
  • When clicked, these links open a new browser tab with the Drivers search page
  • That search page has prefilled Justification, Who the search is for, Reason code (from user's original Vehicles search) and the details of the person (from Vehicles record)

Other changes

  • Service error messages now include a reference ID (which needs to be shared when raising a fault)
  • Update to Global Navigation code (back-end)
  • Back-end fix to justification field error in PDF
  • Backend code update for PDF generation

View release 1.7.0 notes (in LEDS Knowledge Hub)

{# Planned functionality #}

Planned functionality

The list below includes only significant pieces of upcoming functionality. Priorities can change due to user feedback and unforseen dependencies, so please check regularly for updates.

Early 2024

Historic insurance for today's date
Enable users to enter today's date when searching for past insurance (as past insurance can be used for all policing purposes).
Update functionality (1st release)
Ability for users (with the appropriate entitlement) to create and update police records, and add and update police reports.
Basic enquiry entitlement
Offer an entitlement where users will have access to vehicle and keeper details, including status of insurance but without any further insurance details.
Update functionality (2nd release)
Extend Update functionality to deleting and pair/deleting reports, blocking and unblocking records, and adding Pol Exam marker.
Previous vehicle details
Introduce a new page on each record, where previous vehicle details are displayed for that VRM (if any exist). Particularly useful for VRMs that are not currently on a vehicle.
Display motability marker
Introduce a new DVLA marker to indicate a vehicle is part of the Motability scheme

Later in 2024

The following functionality is also planned to be delivered within 2024:
  • Search by VIN
  • Search by Post code
  • Search by partial VRM
  • Advanced search (VODS replacement)
  • Present more DVLA data, such as history of VRMs and Keepers
{# Known issues #}

Known issues

When searching for police vehicles in LEDS, the registered keeper details are blank

DVLA are suppressing the fleet details from LEDS because of how they treat they have to treat restricted data in the new API. We have stressed the importance of accessing this data in LEDS.

DVLA may be able to provide us with a code which will indicate that the record is a police fleet vehicle. We are waiting to hear if this is possible. We hope that this could be done fairly soon.

In order to provide all the vehicle record data for fleet vehicles, they need to do more work. They are currently working on a number of different priorities, including getting ready to deliver the capability for VODS, Partial VRM, Postcode and VIN searches in LEDS. The aim is to get all of that work completed by June 2024, but we are yet to receive a plan. We expect that sometime in late November.
